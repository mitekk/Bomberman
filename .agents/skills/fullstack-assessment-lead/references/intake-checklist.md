# Intake Checklist

Use this checklist at the beginning of each assessment task.

## 1. Brief extraction
- Problem statement in one sentence.
- Required deliverables and submission format.
- Timebox and deadline.
- Hard constraints (tech, libraries, deployment target).

## 2. Acceptance criteria
- Explicit success criteria from the brief.
- Implicit success criteria inferred from interviewer context.
- Minimum viable scope that still demonstrates end-to-end capability.

## 3. Scope boundaries
- In scope features.
- Out of scope features.
- Deferred items to preserve delivery confidence.

## 4. Unknowns and assumptions
- List unknowns that block implementation choices.
- Add explicit assumptions for unresolved unknowns.
- Mark assumption impact as low/medium/high.

## 5. Execution plan snapshot
- Vertical slice definition.
- Priority order after vertical slice.
- Testing strategy by layer (FE, BE, integration, smoke E2E).
